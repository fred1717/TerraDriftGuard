  SUMMARY:
    Security group sg-0a1b2c3d4e5f67890 was modified to allow SSH access from any IP address
      (0.0.0.0/0), creating a critical security vulnerability.
  RISK_ASSESSMENT:
    Critical risk - This configuration exposes SSH services to the entire internet, allowing
      potential attackers to attempt brute force attacks, exploit SSH vulnerabilities, or gain
      unauthorized access to EC2 instances if credentials are compromised. This is a direct pathway
      for lateral movement and data exfiltration.
  LIKELY_CAUSE:
    Likely accidental - Given the security group status shows 'NOT_DISCOVERED', this appears to be
      either a misconfigured automation script, an overly permissive temporary debugging change that
      wasn't reverted, or the security group may have been deleted entirely after the non-compliant
      rule was detected.
  REMEDIATION:
    Immediately restrict SSH access by replacing the 0.0.0.0/0 source with specific IP ranges or
      CIDR blocks for authorized networks only. If the security group was deleted, recreate it with
      proper SSH restrictions. Consider using AWS Systems Manager Session Manager as an alternative
      to direct SSH access.
  TERRAFORM_SNIPPET:
    resource "aws_security_group" "restricted_ssh_sg" {
      name        = "restricted-ssh-security-group"
      description = "Security group with restricted SSH access"
      vpc_id      = var.vpc_id
    
      # Allow SSH only from corporate network and VPN ranges
      ingress {
        description = "SSH from corporate network"
        from_port   = 22
        to_port     = 22
        protocol    = "tcp"
        cidr_blocks = ["10.0.0.0/8", "192.168.1.0/24"] # Replace with actual corporate IP ranges
      }
    
      # Allow all outbound traffic
      egress {
        description = "All outbound traffic"
        from_port   = 0
        to_port     = 0
        protocol    = "-1"
        cidr_blocks = ["0.0.0.0/0"]
      }
    
      tags = {
        Name        = "restricted-ssh-sg"
        Environment = var.environment
        Purpose     = "SSH access with IP restrictions"
      }
    }

remediationPlan:
  SUMMARY:
    S3 bucket 'terradriftguard-public-read-test' has been configured with a bucket policy that
      allows public read access, violating the s3-bucket-public-read-prohibited Config rule.
  RISK_ASSESSMENT:
    HIGH - This creates a critical security vulnerability where sensitive data stored in the bucket
      could be accessed by unauthorized users on the internet, potentially leading to data breaches,
      compliance violations, and exposure of confidential information.
  LIKELY_CAUSE:
    This was likely intentional, possibly for testing purposes given the bucket name contains
      'public-read-test', but may have been left in this insecure state accidentally after debugging
      or development work was completed.
  REMEDIATION:
    Remove or modify the S3 bucket policy to eliminate public read access by removing any statements
      that grant 's3:GetObject' or 's3:GetObjectVersion' permissions to the 'Principal' value of '*'
      or AWS account principals that allow public access.
  TERRAFORM_SNIPPET:
    resource "aws_s3_bucket" "terradriftguard_public_read_test" {
      bucket = "terradriftguard-public-read-test"
      
      tags = {
        Environment = "test"
        Purpose = "drift-detection"
      }
    }
    
    # Explicitly block all public access
    resource "aws_s3_bucket_public_access_block" "terradriftguard_public_read_test" {
      bucket = aws_s3_bucket.terradriftguard_public_read_test.id
      
      # Block public ACLs and bucket policies
      block_public_acls       = true
      block_public_policy     = true
      ignore_public_acls      = true
      restrict_public_buckets = true
    }
    
    # Optional: Define a restrictive bucket policy for internal access only
    resource "aws_s3_bucket_policy" "terradriftguard_public_read_test" {
      bucket = aws_s3_bucket.terradriftguard_public_read_test.id
      
      policy = jsonencode({
        Version = "2012-10-17"
        Statement = [
          {
            Sid       = "DenyPublicAccess"
            Effect    = "Deny"
            Principal = "*"
            Action    = "s3:*"
            Resource = [
              aws_s3_bucket.terradriftguard_public_read_test.arn,
              "${aws_s3_bucket.terradriftguard_public_read_test.arn}/*"
            ]
            Condition = {
              Bool = {
                "aws:PrincipalIsAWSService" = "false"
              }
            }
          }
        ]
      })
    }
